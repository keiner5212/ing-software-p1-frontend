Proyecto de Ingeniería de Software: Sistema de Reserva de Salas

Backlog del Producto

Visión General del Proyecto: Actualmente, el proceso de reserva de salas en el edificio de docentes es manual, utilizando papel y lápiz. Para optimizar este proceso, proponemos desarrollar un sistema de reservas que permita reservar salas de manera más rápida y desde cualquier ubicación. Es fundamental definir claramente los requisitos y funcionalidades para el sistema.

Requisitos del Proyecto:

  1. Desarrollo del backend

    • Usar como gestor de bases de datos PostgreSQL.
    • La aplicacion estara desarrollada con Express.js.
    • Crear una tabla para almacenar información de usuarios que puedan reservar salas. Debe incluir datos básicos como nombre, apellido, email, contraseña y documento de identidad, que admita varios formatos.
    • Crear una tabla para almacenar detalles de las salas, incluyendo descripción, capacidad y piso. Además, se debe indicar si la sala está ocupada con un campo booleano, inicializado en falso por defecto. La capacidad y el piso deben ser valores mayores que cero.
    • Desarrollar una tabla para gestionar los roles de los usuarios. Al registrarse, con una columna para especificar el rol, estableciendo "Docente" como valor por defecto.
    • Diseñar una tabla para almacenar los estados de cada sala, permitiendo especificar su estado.
    • Crear una tabla para almacenar enlaces a las fotografías de las salas.
    • Crear una tabla para gestionar las reservaciones realizadas por usuarios registradores y registrados, incluyendo la fecha de inicio y fin de cada reserva. Se debe indicar un estado para conocer la disponibilidad de una sala. La fecha de inicio no puede ser posterior a la fecha de fin.
    • La información de los usuarios debe guardarse encriptada.
    • Todo debe estar pensado para ser manejado con permisos mediante la tabla de roles.
    • Si una sala se encuentra en estado de reserva, no debe ser posible hacer una reserva a la misma en el periodo de tiempo establecido en la reservación.
    • Si un usuario se ausenta a una sala que ha reservado, obtiene una advertencia, si se llega a 3 advertencias para un usuario se le bloquea el acceso a reservas por si mismo.

  2. Desarrollo de la Página Web

    • La página debe ser accesible desde cualquier navegador web.
    • Utilizar Node.js como entorno de trabajo y Angular como framework.
    • Emplear TypeScript en el desarrollo de la página
    • Utilizar Bootstrap para la creación de la página.
    • Incluir un menú de registro para nuevos usuarios, capturando información básica que se almacenará en la base de datos.
    • Incorporar un menú desplegable con secciones para Requerimientos, Equipo SCRUM y SWEBOK.
    • Implementar un apartado de inicio de sesión para que los usuarios puedan acceder con sus credenciales, verificándolas con la base de datos.
    • Permitir a los usuarios reservar salas, ya sea para sí mismos o para otros usuarios por parte de administradores.
    • Diseñar una interfaz web intuitiva para docentes de diferentes edades y personal del edificio.
    • Incluir un apartado que presente información sobre los desarrolladores del sitio.
    • Exhibir el ciclo de vida del software en la página.
    • Contar con una sección para visualizar cada área de conocimiento en el SWEBOK.
    • Incorporar una sección para ver los requerimientos del proyecto.
    • Desarrollar un menú de registro para usuarios nuevos, donde se registre con información basica la cual se almacenara en la base de datos.
    • Desarrollar un apartado de inicio de seción para verificar las credenciales de cada usuario.
    • Mostrar la estructura del equipo SCRUM desde la página.
    • Desarrollar una sección para mostrar las salas disponibles.
    • Garantizar una interacción fluida entre el usuario y el sistema de reservas.
    • Los usuarios con rol de administrador podran ver toda la informacion.
    • El inicio de sesion para administradores estara separado de los usuarios.
    • Los usuarios con rol de administrador podran modificar (actualizar, borrar, etc) toda la informacion.
    • Los usuarios podran reservar una sala por un tiempo determinado.
    • Los usuarios podran cancelar sus reservas.
    • Los usuarios podran ver el historial de sus reservas.
    • Los usuarios podran revisar cuantas advertencias llevan.
    • Un usuario tiene un limite de reservas semanales de 3.

  3. Optimización del Proceso de Reserva

    • Facilitar la reserva rápida desde cualquier ubicación.
    • Garantizar eficiencia en el proceso de reserva de salas.
    • Implementar notificaciones para confirmación de reservas.

  4. Seguridad y Privacidad

    • Integrar medidas de seguridad para proteger la información de los usuarios (encriptación, autenticación, etc).
    • Implementar autenticación segura para acceder al sistema.
